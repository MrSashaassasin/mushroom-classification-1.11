# Определение съедобности грибов с помощью машинного обучения и MLOps

Итоговый аттестационный проект в рамках программы «ИИ. Алгоритмы машинного обучения на языке Python».

**Выполнил:** Малахов Александр Александрович  
**Группа:** 424-1  

---

## Ключевые особенности проекта
* **Полный цикл разработки (End-to-End):** от детального EDA и фичеринжиниринга до развертывания модели.
* **MLOps и трекинг экспериментов:** интеграция с **MLflow** (SQLite-бэкенд) для логирования параметров, метрик и версионирования моделей в Model Registry.
* **Проверка на переобучение:** построены кривые обучения (**Learning Curves**) для анализа стабильности модели при масштабировании данных.
* **Интерпретируемость уровня Enterprise:** использование **SHAP (Summary & Waterfall plots)** для глобального и локального объяснения предсказаний модели.
* **Готовый REST-API сервис:** высокопроизводительный сервис на **FastAPI** с валидацией входных данных через Pydantic.

### Анализ качества:
* **Recall на тестовой выборке составляет 100%**. Для данной задачи минимизация ошибок False Negative (пропуск ядовитого гриба) является критически важной для жизни пользователей.
* **Анализ кривых обучения (Learning Curves)** показал идеальное схождение метрик на обучении и валидации с разрывом $< 0.01$, что строго доказывает **отсутствие переобучения** (Overfitting) или недообучения (Underfitting).
* **Анализ ошибок (Error Analysis)** подтвердил нулевой уровень ошибок на тестовом множестве благодаря четкой линейной и нелинейной разделимости классов по морфологическим признакам (ключевую роль играет признак `odor` — запах).

## Архитектура репозитория
* `mushroom_classification.ipynb` — интерактивный Jupyter-ноутбук (полный цикл: EDA, Feature Engineering, Обучение, MLflow, SHAP, Валидация)
* `app.py` — REST-API веб-сервис на базе веб-фреймворка FastAPI
* `mlflow.db` — SQLite база данных с историей всех запусков MLflow
* `mushrooms.csv` — исходный набор данных UCI
* `mushroom_model.pkl` — сохраненная лучшая модель Random Forest
* `label_encoders.pkl` — сохраненные объекты LabelEncoder для инференса
* `feature_names.json` — строгий список входных признаков для API
* `requirements.txt` — зафиксированные версии зависимостей
* `*.png` — визуализации графиков для отчета

## Инструкция по локальному запуску REST-API сервиса

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/MrSashaassassin/mushroom-classification.git
   cd mushroom-classification
   ```
   
2. **Установите зависимости:**
  ```bash
  pip install -r requirements.txt
  ```

3. **Запустите сервер FastAPI:**
  ```bash
  uvicorn app:app --reload
  ```

4. **Документация API:**
    Интерактивная документация Swagger UI и отправка тестовых запросов доступны по адресу: http://127.0.0.1:8000/docs
